## What was verified

- **Contact:** valid submit → `?sent=1`; invalid email / empty blocked; honeypot
  silently ignored; throttle returns 429 after 5/min; emoji + SQL-fragment + 4k text
  stored without error (escaped).
- **Order flow:** form → checkout shows totals; terms-unchecked blocks confirm;
  full confirm → `/checkout/success` with invoice generated; invalid referral code
  does not crash.
- **Token portals (new code):** brief loads with hex-64 token; brief submit sets
  `brief_submitted_at` + redirects to thanks; **re-visit short-circuits to thanks**
  (submit-once gate); progress page shows seeded ProgressNote (new relation manager);
  testimonial + referrer gates load; malformed/missing token → 403; invalid
  referrer code → 403.
- **Invoice:** HTML page 200; PDF returns `application/pdf` non-empty; unknown
  invoice → 404.
- **Chat (live OpenAI):** identity question short-circuits (no API spend); empty
  message → 400 RC-CHAT-001; live question returns non-empty reply.
- **Admin:** login; all 13 Filament resource list pages render; AuditLog page renders;
  dashboard widgets load — no Server Error / Whoops.
- **New schema:** `order_payments` FK cascade verified live (delete Order →
  payments auto-deleted, separate tinker test).
- **Resilience:** mid-submit network kill on contact + order keeps user on form
  (no data loss / crash); slow link still completes order.
